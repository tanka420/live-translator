# Recording Plan

## Capture Order

1. Setup shot
   - Show the project folder.
   - Show `.env.example`, then mention `.env` privately contains the API key.
   - Run `npm install` and `npm run dev`.

2. App first look
   - Show `Live Event Translator` title.
   - Show the four event chips: English speech, Korean interview, Japanese event,
     Chinese keynote.
   - Show `Vietnamese` selected by default.
   - Show audio mix at 85% translated.

3. Browser permission
   - Open a source tab with official/public event audio.
   - Click `Choose event tab`.
   - Pick the source tab in the browser picker.
   - Make sure `Share tab audio` is enabled.

4. Main proof
   - Use one clear English event clip: a public speech, Fed/central bank speech,
     tech keynote, or press conference.
   - Capture translated Vietnamese audio and translated transcript.
   - Keep a short split-screen moment: source tab on one side, app on the other.

5. Global proof montage
   - Capture two or three short tests:
     - Korean interview or live event.
     - Japanese public event or product demo.
     - Chinese keynote or product launch.
   - Keep each proof segment short enough for editing.

6. Transcript proof
   - Zoom in on the translated transcript while the source audio continues.
   - Capture a realistic latency moment instead of hiding every delay.

7. Product framing
   - Record one clean still/slow pan of the UI with the translated transcript.
   - Record the stop button ending the session.

## Shot List For Long Video

- Cold open: finished app translating a live event into Vietnamese.
- Problem: live events and interviews often arrive before Vietnamese summaries.
- Build: server creates client secret; browser uses WebRTC; app plays translated
  speech and transcript.
- Tests: English, Korean/Japanese, Chinese.
- Honest close: useful for understanding live content, not a replacement for
  professional interpreting.

## Short Trailer Cuts

- Hook: "Mình vừa build một app nghe sự kiện quốc tế bằng tiếng Việt realtime."
- Visual: source event tab plus translated transcript.
- Proof: audio meter moving, WebRTC connected, Vietnamese text appearing.
- CTA: "Video dài hướng dẫn build từ đầu đã có trên YouTube."

## Quality Checklist

- API key is never visible.
- Source clip is from an official/public channel when possible.
- Browser picker shows tab audio enabled.
- Translated voice is audible over source audio.
- Captions/transcript are readable at 16:9 and crop-safe for 9:16.
- No claim that translation is perfect or that this replaces human interpreters.
